## 6.2 Abstract (Binary) Relations
### 6.2.1 Definition
Let's jump right in and start talking about relations. We'll give you a definition and then a bunch of examples.
<div class="def-definition" markdown>
**Definition 6.2.1. Let A, B be sets. A relation between A and B is a set of**
</div>
ordered pairs, $R \subseteq A \times B$. Given elements $a \in A$ and $b \in B$, we say $a$ and $b$ are related if and only if $(a, b) \in R$. The set $A$ is called the domain and the set $B$ is called the codomain. The set $R$ is called the relation set. If $A = B$, we say $R$ is a relation on $A$. It is also fairly common to write $x \, R \, y$ to mean $(x, y) \in R$. When we have defined a relation in this way, we will stick to the notation $(x, y) \in R$ to indicate the underlying set structure. Later on, we will sometimes define relations by using some symbol, like $x < y$ or $x \star y$, and so on.
Remark 6.2.2. A relation, as we defined it here, is also sometimes referred to
as a binary relation. This is because there are two "inputs" of the relation; the set $R$ consists of ordered pairs. We could generalize this idea to ternary relations. That is, given sets $A, B, C$, we could define a set $R \subseteq A \times B \times C$ to be a ternary relation and say $a, b, c$ are related if and only if $(a, b, c) \in R$. We could generalize this further to relations with $n$ "inputs", as well. In this context, though, we will only consider binary relations, so we will use the term relation to mean binary relation.
Remark 6.2.3. A relation R is often defined by identifying a property of elements
of $A$ and $B$ (phrased as a variable proposition $P(a, b)$) and setting $(a, b) \in R \Leftrightarrow P(a, b)$
Examples
Example 6.2.4. Let W = {English words} and L = {English letters}. Define
the relation $R$ by setting $(w, \ell) \in R \Leftrightarrow w$ begins with $\ell$. Then, $(\text{mathematics}, m) \in R$ and $(\text{golf}, g) \in R$ because these are valid words and we have identified their starting letters. For some non-examples, notice that $(\text{knowledge}, n) \notin R$ and $(\text{you}, u) \notin R$. Furthermore, note that $(\text{zyzyxyqy}, z) \notin R$ because $\text{zyzyxyqy} \notin W$. It is often the case that $A = B$, so $R$ defines a relation on pairs of elements from one set. The next example considers this situation.

> 🇨🇳 ## 6.2 抽象（二元）关系 ### 6.2.1 定义 让我们直接开始讨论关系。我们先给出定义，然后给出一组例子。
**定义 6.2.1.** 设 $A$、$B$ 为集合。$A$ 与 $B$ 之间的关系（Relation）是一个有序对的集合 $R \subseteq A \times B$。给定元素 $a \in A$ 和 $b \in B$，当且仅当 $(a, b) \in R$ 时，我们说 $a$ 与 $b$ 是相关的。集合 $A$ 称为定义域（Domain），集合 $B$ 称为陪域（Codomain）。集合 $R$ 称为关系集。若 $A = B$，我们称 $R$ 是 $A$ 上的关系。另一种常见的写法是 $x \, R \, y$，表示 $(x, y) \in R$。当我们以这种方式定义关系时，我们将坚持使用记号 $(x, y) \in R$ 来表示底层的集合结构。之后，我们有时也会使用某个符号来定义关系，如 $x < y$ 或 $x \star y$ 等。
**注 6.2.2.** 我们这里定义的关系有时也称为二元关系（Binary Relation）。这是因为关系有两个"输入"；集合 $R$ 由有序对组成。我们可以将此概念推广到三元关系。即，给定集合 $A, B, C$，我们可以定义集合 $R \subseteq A \times B \times C$ 为三元关系，并说当且仅当 $(a, b, c) \in R$ 时 $a$、$b$、$c$ 是相关的。我们还可以进一步推广到具有 $n$ 个"输入"的关系。不过在当前语境下，我们只考虑二元关系，因此我们将使用"关系"一词来指代二元关系。
**注 6.2.3.** 关系 $R$ 通常通过识别 $A$ 和 $B$ 的元素所具有的某个性质（表述为变量命题 $P(a, b)$）来定义，并设定 $(a, b) \in R \Leftrightarrow P(a, b)$。
**例子 6.2.4.** 设 $W = \{\text{英文单词}\}$，$L = \{\text{英文字母}\}$。定义关系 $R$ 如下：$(w, \ell) \in R \Leftrightarrow w$ 以 $\ell$ 开头。则 $(\text{mathematics}, m) \in R$，$(\text{golf}, g) \in R$，因为这些都是合法的单词且我们识别了它们的首字母。对于一些反例，注意 $(\text{knowledge}, n) \notin R$，$(\text{you}, u) \notin R$。此外，注意 $(\text{zyzyxyqy}, z) \notin R$，因为 $\text{zyzyxyqy} \notin W$。通常 $A = B$，此时 $R$ 定义了同一个集合中元素对之间的关系。下一个例子考虑了这种情况。
